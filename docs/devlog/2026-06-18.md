## 修复

- **互动轨迹浮点 id**：全站流 UNION 转发行返回的帖子 id 在某些路径以浮点形态（`"3193.0"`）返回，InteractionSystem/CascadeSystem 原样写入轨迹 `target_post_id`，与帖子块整数 id 对不上 → 检视器对互动块显示无轨迹。新增 `simulator/src/ids.ts` 的 `idStr` 规整为整数串（发轨迹与调 API 前）；历史脏数据另写 `scripts/fix-trace-ids.mjs` 一次性订正（demo 修 1013 行）。按「不为早期 bug 写兼容代码」原则，**不在查询侧加容错**。
- **编辑器后端 better-sqlite3 ABI 不匹配（真因）**：决策轨迹「为什么」一直读空，并非轨迹没记（库里 3715 条）也非 `.0`，而是编辑器后端经 Electron utilityProcess 运行、其 Node ABI（130）与为系统 node（127）编译的 `better-sqlite3.node` 不匹配，`new Database()` 抛 NODE_MODULE_VERSION 错被 trace-reader 静默吞掉。改为 `child_process.fork` + 系统 node（`execPath=npm_node_execpath`）拉起编辑器后端，三进程统一 ABI、免 electron-rebuild；trace-reader 用 `existsSync` 区分"世界无库"（静默）与真实读失败（打日志，不再静默吞错）。打包 M5-6 再改为「为 Electron 重建原生模块 + utilityProcess」。

## 账号驱动模型对齐与落档

与用户讨论对齐了账号模型（偿还 memory `account-model-realign` 的债），结论落为新文档 `docs/m5-account-model.md`：

- **账号类型即默认驱动模式，是运行时模式而非固定身份。** 五命名预设——真人 / 纯 LLM / 半 LLM / 纯模拟器 / 一次性氛围——展开为四条正交轴（归属 / 内容来源 / 互动来源 / 内态）+ 一条降级地板轴（`degrade` 退档 / `silent` 闭嘴）。
- **真人账号是唯一铁墙**，GM 与模拟器永不触碰；其余账号 GM 可临时升级（沿用 GM 唤醒机制）、资源不足时按降级地板下压。实际驱动者按动作运行时解析，不写进账号字段。
- **状态机是贯穿所有非真人账号的底座**：mood / 关系 / 记忆数值一律维护，驱动模式只决定谁消费这些数值产出动作。
- **纯 LLM 退出确定性互动层**：引擎绝不替核心 NPC 自动点赞 / 转发，互动 / 级联系统须维护"自动操作资格"白名单。这是用户提出的——皇冠级 NPC 点错一个赞都算 OOC，故 `silent`：宁可不出现，也不出 OOC。
- **分库映射**：真人仅 world.db；持久被驱动账号 world.db + npc-state.db + sim-trace.db；一次性氛围号 world.db + sim-trace.db 但无 npc-state 行；rp.json / lore.md 仅 LLM 档账号需要。
- 落地点为 npc-profile 字段 + 建号流程 + NPC 设计器（路线图 1.7）。

同步对齐了 `m5-design.md`（账号一节由"三档"改为五预设驱动模式）、`m5-x-re-plan.md`（账号模型约束加驱动模式）、`m5-npc-state-machine.md`（状态机底座消费者随模式变）、`m5-x-roadmap.md` 1.7（NPC 设计器加驱动模式 / 降级策略字段）、`m5-x-phase1-baseline.md`、`m5-x-index.md`、`CLAUDE.md`。本次仅文档，无代码改动。
