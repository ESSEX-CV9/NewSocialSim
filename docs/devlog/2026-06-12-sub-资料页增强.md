## 认证体系（migration v9/v10）

- `users` 加 `verified`（none/personal/org）与 `verified_at`（认证状态变化时按模拟时间打点）。`UserSummary` 全部组装点（帖子作者/搜索/时间线转发者/关注列表/通知 actor/推荐关注/共同关注者）带 `verified`，徽标（`ri-verified-badge-fill`，新主题令牌 `x-gold`，徽标色不随主题变）铺设到全站所有用户名显示点。
- 入口：未认证的本人页在名字旁显示 X 式"获得认证"药丸按钮，点击下拉直选个人/组织认证即时生效；已认证后悬停徽标 400ms（或点击）弹 X 式说明卡（认证类型/自某年月通过/ID 已认证），本人卡底部"管理认证"进三选弹窗。悬停关闭有 300ms 宽限（移入卡内不消失）。

## 资料页字段扩展（migration v9/v10）

- `users` 加 `website`（存储时无协议自动补 https://）、`location`（自由文本地名，可虚构）、`birth_date`（YYYY-MM-DD）、`profession`（类别 key，前端 `i18n/professions.ts` 24 项双语映射）。
- 编辑资料弹窗新增四字段；资料页元信息行按 X 顺序：类别/位置/链接/出生于/加入时间；简介改用 PostContent 解析（@/URL/#话题 变蓝可点）。
- 资料响应加 `knownFollowers`（观察者关注的人里也关注此人的前 3 个 + 总数），资料页粉丝数下方小头像堆叠 + 社交证明文案，点击跳粉丝列表。

## 链接卡片修复（ecf5ac8）

四个叠加根因：①失败结果永久缓存从不重试（一次网络抖动即该 URL 永远无卡）→ 失败条目在下次有帖引用时重试，成功条目仍永久缓存；②YouTube og 标签在 ~645KB 处而读取限额 512KB → 放宽至 2MB、超时 5s→10s；③无浏览器 UA 部分站点回机器人页 → 改 Chrome UA；④B 站 og:image 自带 `@WxH` 低清后缀 → hdslb.com 剥后缀取原图（实测 17KB→1MB）。附一次性脚本 `scripts/clear-stale-link-cards.mjs` 清除存量失效缓存（已对现代地球执行）。

## 遇到并解决的问题

- React StrictMode 的"挂载→清理→再挂载"暴露资源生命周期错位：objectURL 建在渲染期（useState 初始化器）、revoke 在 effect cleanup，被双挂载提前撤销导致裁剪器图片永远加载不出。资源创建必须放进 effect 并与清理配对。
- 悬浮类 UI（认证说明卡/用户悬浮卡）锚点与卡片之间有空隙，鼠标穿越瞬间触发 mouseleave 即关——统一改为"开 400ms 延迟 + 关 300ms 宽限"模式。
- GIF 经 canvas 裁剪必然失去动画，与"可裁剪"二选一：GIF 选作头像时跳过裁剪直传原图（横幅仍裁剪出静帧）。
- PS 5.1 Invoke-RestMethod 发中文 JSON body 乱码再次出现（写入 location 变 "???"），验证脚本侧问题，浏览器不受影响。
