## 关键改动

- **互动状态缓存写穿**（`7ddfc25`）：根因是 PostCard 把互动状态 `useState(post.xxx)` 拷贝为本地 state、mutation 后从不写 react-query 缓存。新增 `client/src/api/postCache.ts`：按 queryKey 前缀清单对三种缓存形态（InfiniteData 的 TimelineItem/PostView 列表、`{post}` 单帖包装）做不可变写穿，含 `.quoted`/`.inReplyTo` 嵌套；赞/转发/书签/关注/引用数/回复数/删帖全部走写穿，成员资格变化另行 invalidate（书签用 active 立即移除行，其余 `refetchType:'none'` 防活跃列表重拉跳动）。ProfilePage 关注按钮与推荐关注卡同步纳入。新增"含 PostView 的查询须登记前缀"的约定（postCache.ts 文件头注释）。
- **视频体系**（`055ebbb`/`ee234d7`/`a3d150a`）：新增 `client/src/components/videoPlayback.ts`——按媒体 URL（含 ?w= 天然防跨世界串号）的会话级进度表 + "play 事件接管单活跃播放"注册器。帖内视频：50% 进视窗静音自动播放、出视窗暂停、新视频起播自动停旧的（对齐 X）；点画面经透明覆盖层（底部留 56px 给原生控制条）放大到查看器有声续播，播放暂停只走控制条；查看器/原生全屏内保持默认点击行为。MediaLightbox 支持视频与"查看帖子"入口、k/N 指示。
- **回复对话串**（`3817b8d`）：`GET /api/users/:handle/posts?type=replies` 每条回复嵌入一层被回复帖（`PostView.inReplyTo`，repo 新增 findVisibleByIds 复用屏蔽/隐藏 SQL 片段过滤）；不可见（已删/被屏蔽/被隐藏）时降级"回复 @handle"蓝标行（`replyToHandle` 从软删行仍可取）。前端 PostCard 加 threadTop（头像连接线+去底边框）与 replyToFallbackHandle。
- **媒体规则**（`e658194`）：一帖上限 4 → 20（图/视频共享配额、可混排），限制点共三处——media.service 的 MAX_PER_POST、发帖路由 schema maxItems（首轮调研漏掉，API 验证时暴露）、Composer 的 MAX_MEDIA。宫格混排视频格为首帧+播放角标（不自动播放，避免与单活跃规则打架）；超 4 个第 4 格叠 +N；Composer 多媒体预览改横向滚动缩略条。
- **@ 候选与输入高亮**（`e9b8a70`）：高亮镜像层在文档流中撑高度、textarea 透明文字绝对覆盖且禁内部滚动（不存在 scroll 同步错位问题）；高亮口径与 PostContent 一致（URL/#话题/@handle）。@ 触发候选：光标前正则（@ 前必须是行首或非 handle 字符防 email 误触发），前缀搜索 300ms 防抖、裸 @ 给推荐关注，最多 5 个，↑↓/Enter/Tab/Esc/鼠标交互。
- **编辑资料弹窗**（`ee70001`）：新组件 ImageCropper（原生 Canvas 零依赖：cover 基准 1-4x 缩放、指针拖动钳制、滚轮+滑杆，头像 1:1 圆形遮罩出 400×400、横幅 3:1 出 1500×500，GIF 出静帧为已知取舍）与 EditProfileModal（X 式布局，选图进裁剪、恢复默认、保存写 auth 快照并失效 profile 缓存）；删除旧内联表单。资料页头像/横幅有图时点击经 MediaLightbox 放大。
- 其余：搜索/书签头像 SQL 漏列修复（buildViews 读 `author_avatar_media_id`，undefined 通过 `!== null` 拼出必 404 URL）；`scrollbar-gutter: stable` 消除切页抖动；通知列表 `refetchOnMount:'always'` + 30s 轮询与未读徽标同节拍；通知卡改"卡片点击导航+内部真实链接"（原整卡 `<a>` 嵌链接为非法 HTML）。

## 遇到并解决的问题

- 发帖路由 JSON schema 的 `maxItems: 4` 未被首轮调研发现，5 图发帖被 schema 层 400 才暴露——多层校验（schema/service）改同一规则时要全局 grep 数字本身。
- 软删帖保留 post_media 关联，验证中误挂到测试帖的媒体永久占用（"媒体已被其他帖子使用"），重测需新上传媒体。
- 高亮镜像层若用传统"textarea 滚动 + scrollTop 同步"方案，内部滚动条会改变排版宽度导致换行错位；改为镜像层在文档流撑高、textarea 绝对覆盖禁滚动后从结构上消除。
