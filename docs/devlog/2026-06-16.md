## 方法论与架构对齐

回退根因：上一轮以完整 NPC 状态机为目标直接铺开，唯一产出物是网站上一条与普通帖无异的帖，模拟器选了哪个池、是否区分帖与回复、状态机是否生效全程不可识别——每阶段对用户都是黑箱。本轮原则写入 `docs/m5-x-re-plan.md`：

- 先确定性后 LLM：tick 层做成零 LLM 纯规则，LLM Agent 路径先移出关键路径。
- 编辑器为唯一观察窗，与模拟器同里程碑同步交付。
- 三条数据线分离：改世界走 server API、观测世界走编辑器后端、配置随世界文件夹。
- 编辑器重建为多窗格 IDE 形态 + 独立 Fastify 后端（现在就上）。
- 决策轨迹即时间轴块（账号轨道 × 模拟时间），落盘 per-world JSONL。
- 四步阶梯：Step 0 地基 → 1 顶层帖 → 2 配图 → 3 回复，行为状态机后置。

## 内容池模型统一为 ECS 三层

内容池由扁平 string[] 重设计为与模拟器 ECS 同构的三层：组件类型（自带候选片段）→ 语法（有序引用组件类型、每槽可选或带概率）→ 池（维度定义 + 引用哪几套语法）。槽位平等，颗粒度由作者自由切分；占位符即内联组件引用。`docs/m5-x-re-plan.md` 写入完整模型，`docs/m5-npc-state-machine.md` 旧 PoolEntry 完整整句 schema 与选择算法随之迁移，`docs/m5-real-usage-contract.md` 相关引用对齐，三份文档收口一致。

## Step 0a：代理建号

新增 `POST /api/admin/users`：创建 `is_bot=1` 虚拟账号，校验 handle 拟真、拒绝暴露虚拟身份的命名约定（`sim_` / `_amb` / `bot` / `npc` / 通用 stem+序号枚举），未提供密码时随机生成并在响应中返回供登录。配套 `ApiClient.adminCreateUser`。冒烟测试：拟真建号 201 且 is_bot=1、重复 handle 409、命名违规 400。

## Step 0b：模拟器跟随活动世界

账号驱动改为不存密码模型：新增 `POST /api/admin/login-as`，凭 admin key 为账号换一张 JWT 登录票，模拟器走与真人完全相同的用户端点驱动，任何地方不存明文密码。被驱动账号 = 有 npc 档案者（身份取自数据库、驱动配置取自 npc-profiles.json）。roster 文件方案一度落地（0b-i）后即被此票据方案取代——账号已在数据库、roster 文件冗余且需存明文密码。

模拟器由固定 registry 的 TickEngine 重构为编排器 `simulator.ts`：启动只读基础设施（API 地址 / admin key / tick 间隔），运行时查 `GET /api/admin/worlds/active` 跟随活动世界；世界变更则 flush 旧世界、按新世界档案重登账号、重建系统。时间基修正——tick 在世界模拟时间下运行（按流速本地推算、暂停即冻结），不再混 `Date.now()`。发帖确定性化——PostingSystem 去掉 LLM Agent 路径只走模板/池。顺带修 `ApiClient.createPost` 返回解析（`{post:{id}}` → `{id}`），消除级联接楼取 id 踩空。删除 `tick-engine.ts`。

## 账号模型前端对齐：移除虚拟用户标记

前端在帖子卡、个人主页、关注列表三处据 `isBot` 渲染"虚拟用户"徽章，破坏沉浸、违反"网站侧所有账号一视同仁、不按是否虚拟做区分"。先移除三处徽章与 i18n 文案 `profile.bot`，并将 `design.md` / `m5-design.md` 中"前端可选择是否显示 bot 徽章"的旧授权改写为"前端一律不得据此区分或标记账号"。

继而把 `isBot` 从公共数据层彻底删除：删去公共类型 `User` / `UserSummary` 的该字段，及 server 中拼装这两个 DTO 的全部 11 处（follows / posts / messages×3 / notifications / search / timeline / users×2 / users.repo）与前端 `AuthContext` 映射。`is_bot` 数据库列与管理端用途（建号设 is_bot=1、admin 列用户、模拟器据 npc 档案驱动）不变，公共 API 响应不再含 isBot——前端从数据层即无从区分账号真伪。少数 SQL 查询仍顺带 SELECT `is_bot`（查出不用、对前端不可见），暂保留以避免改 SQL 字符串无类型兜底的风险。

验证：typecheck 全绿、前端构建通过；GET 时间线 / 用户主页响应的作者字段仅含 `id / handle / displayName / avatarUrl / verified`。
