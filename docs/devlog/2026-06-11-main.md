## 各里程碑详情

### M1 骨架与核心设施

- 初始化 git 仓库与 npm workspaces monorepo（shared / server / client / simulator 四个工作区）。
- shared 包：World、User、Post 等纯数据实体类型。
- server 核心设施（`server/src/core/`）：
  - SimClock 模拟时钟：模拟时间 = 锚点模拟时间 + 真实流逝 × 流速，支持调速、暂停、直接设定时间；
  - SQLite 连接封装与基于 `PRAGMA user_version` 的版本化 migration 运行器；
  - WorldManager：世界创建/枚举/热切换，时钟状态持久化于各世界的 world.json，活动世界 id 记录在 data/state.json，重启自动恢复；
  - 统一业务错误类型（AppError 及子类，由应用级错误处理器映射为 HTTP 状态码）。
- worlds 模块：世界列表 / 创建 / 激活 / 查询当前的 API。
- 验收通过：创建 60 倍速与 1 倍速两个世界，热切换前后模拟时间独立推进；被切换出去的世界时钟冻结，切回后从挂起点继续。

### M2 用户系统与认证

- core/auth：JWT 签名密钥首次启动生成后落盘复用；requireAuth 守卫校验 token 并核对其 worldId 与当前活动世界一致。
- auth 模块：注册 / 登录 / me，密码用 Node 内置 `crypto.scrypt` 哈希（无原生依赖）。
- users 模块：公开主页（含粉丝/关注/帖子计数）、编辑个人资料。
- 升级 `@fastify/jwt` 9 → 10：v9 依赖的 fast-jwt 存在 6 个已知漏洞（含算法混淆与鉴权绕过），升级后 `npm audit` 为 0 漏洞。
- 验收通过：重复注册 409、错误密码 401、热切换世界后旧 token 401、跨世界数据完全隔离（世界 B 查不到世界 A 的用户）。

### M3 内容与互动

- posts：发帖 / 回复 / 引用 / 详情 / 回复列表 / 用户帖子列表。删除为软删除：帖子变为墓碑（内容清空、deleted=true），对话串不断链，墓碑不出现在时间线与搜索中。
- interactions：点赞 / 转发为幂等开关，反规范化计数随事务维护。
- follows：关注 / 取关 / 粉丝列表 / 关注列表，禁止关注自己。
- notifications：reply / quote / like / repost / follow 五类通知、未读数、一键全部已读；用户对自己的操作不产生通知。
- timeline：关注流（自己与关注者的原创帖 + 转发条目，转发按转发时间排序并标注转发者）与全站流（匿名可访问）；统一游标分页（base64url 编码的排序键数组）。
- search：帖子 / 用户子串搜索（LIKE 实现，转义了通配符）。
- 新增 migration v2：posts 表加 deleted 列。
- 端到端验收脚本 `scripts/verify-m3.ps1`：26 项断言全部通过，覆盖发帖→关注→回复→引用→赞→转发→时间线→通知→搜索→删除全流程。

## 当日遇到并解决的问题

- Fastify 5 的 `setErrorHandler` 参数默认类型为 unknown，显式标注 `FastifyError | AppError` 后通过类型检查。
- 应用级错误处理器曾把 415 等 Fastify 协议错误统一映射为 500，掩盖了真实状态码；改为非 5xx 状态码原样透传。
- TypeScript `exactOptionalPropertyTypes` 下，路由选项同时含 preHandler 与字面量 schema 时 Fastify 无法推断 handler 泛型；在路由方法上显式声明泛型解决。
- M3 验收一度出现 5 项断言失败，排查确认服务端全程正确，问题全部在 PowerShell 5.1 环境：
  - 无 BOM 的 UTF-8 .ps1 文件被按 GBK 解析（中文内容全部乱码导致语法错误）；
  - `$home`、`$global` 是 PowerShell 内置只读/特殊变量，不可用作脚本变量名；
  - 单个 PSCustomObject 的 `.Count` 返回空（该内置成员 PowerShell 6 才引入），需用 `@()` 包装为数组后再取 Count。
- `Set-Content -Encoding utf8` 写出的 state.json 带 BOM，`JSON.parse` 直接报错导致服务器无法启动；除重写该文件外，WorldManager 读 JSON 一律先剥离 BOM。
- Vite 6 默认绑定 localhost（解析为 IPv6 ::1），用 127.0.0.1 探测连不上，用 localhost 访问正常。
