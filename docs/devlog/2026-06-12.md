## 视觉体系

- **设计令牌对齐 X**：纯黑底 #000、卡片 #16181c、边框 #2f3336、次级文字 #71767b，三强调色严格分工（蓝 #1d9bf0 操作 / 绿 #00ba7c 转发 / 粉 #f91880 赞）；Inter 字体、15px 正文、4px 基数尺寸、0.15/0.2s 统一过渡。令牌以 Tailwind 4 `@theme` 定义（`bg-x-*` 等工具类）。
- **图标库两次迁移**：先从 emoji 全量替换为 FontAwesome（npm 本地引入）；随后因 FA 免费版缺少房子/放大镜/地球的空心变体、无法实现导航"未选中空心/选中实心"双态，整体迁移到 Remix Icon（同为 npm 离线方案，全部图标具备 -line/-fill 双态）。
- **主题系统抽象**（commit `b585ac4`）：颜色改为两层结构——`[data-theme]` CSS 变量块定义主题，`@theme inline` 把 `x-*` 令牌映射到变量，全部工具类自动跟随主题。内置深色/浅色（浅色按 X 原版色值），ThemeContext 持久化到 localStorage。新增主题只需在 index.css 加一个变量块并在 theme/themes.ts 注册。
- 通用元素：毛玻璃 sticky 顶栏（rgba 背景 + blur(12px)，背景色随主题）、Tab 底部 56×4px 蓝色指示条、旋转加载圈、48px 图标空状态、互动按钮 hover 同色 10% 圆形气泡 + 整卡 3% 悬停叠加。

### 浏览量（migration v5）

- posts 表新增 `view_count` 列（无回填，历史帖从 0 起算）。计数口径为曝光：帖子卡片 50% 进入视口算一次；同一页面会话内每帖只计一次，刷新后再看再计。
- 新增公开接口 `POST /api/posts/views`（optionalAuth，body `{ ids: 1–100 个唯一正整数 }`，返回 204）：批量曝光 +1，已删除与不存在的 id 静默忽略。挂 posts 模块——浏览无 (user, post) 行，不属于 interactions 的开关形态。

- 前端新增 `useViewTracking`：模块级共享 IntersectionObserver（threshold 0.5）+ 会话级已报 Set + 待报队列，2 秒 debounce 或满 20 条批量上报，页面隐藏时 `sendBeacon` 兜底；墓碑帖与引用嵌入卡不观察；详情页大卡走同一 PostCard 自然计入。世界热切换不刷新页面而帖子 id 空间重叠，`WorldsPage.activate()` 中调用 `resetViewTracking()` 清空会话状态。
- PostCard 互动行在赞与书签之间新增浏览量（`ri-bar-chart-2-line`，蓝色 hover，点击仅 stopPropagation 无动作，0 时只显示图标）。

### 本周期遇到并解决的问题

- 用户验收发现帖子图片全裂、发帖框预览不显示：`GET /api/media/:id/file` 返回 200 但 body 为 0 字节。根因是 Fastify 5 的 async handler 竞争——handler 调用 `reply.send(stream)` 后未 `return reply`，async 函数 resolve 出的 undefined 被当作响应载荷、抢在流管道前以空 body 结束响应。JSON 接口同步序列化不受影响，**流式响应必须 `return reply.send(stream)`**。隔离复现确证后一行修复。验证教训：流式接口的断言必须包含 body 字节数，只看状态码与响应头会漏掉此类问题。
- 修复后浏览器仍可能显示裂图：空响应已按 `immutable` 缓存头被缓存，需 Ctrl+F5 强刷一次。

### 视频支持

- 上传支持 mp4/webm，上限 100MB（multipart 全局限额同步上调；图片的 10MB 限制在 service 层单独校验）。视频走流式落盘不进内存：先写 `pending-<pid>-<计数>` 临时文件，入库取 id 后改名为 `<id>.<ext>`；multipart 限额触发时流被静默截断，落盘后显式检查 `truncated` 与字节数。宽高存 NULL（不引入视频探测依赖）。
- 文件端点支持 HTTP Range：`bytes=a-b` 与 `bytes=-N` 后缀形式均解析，返回 206 + `Content-Range`/`Accept-Ranges`，不合法的 Range 按整文件 200 响应；视频拖进度条依赖此能力。
- 挂帖规则：视频只能单独 1 条、不与图片混排（服务端 `validateAttachable` 校验，Composer 选择阶段同样拦截并提示）。
