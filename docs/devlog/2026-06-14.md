# 开发日志 2026-06-14

## 概要

M5 模拟器第一轮实施：完成 M5-1 至 M5-4 四个里程碑，搭建模拟器骨架、世界管理全套 API、话题与内容引擎、LLM Agent 接入。新增 simulator / editor 两个工作区，server 新增 admin 模块（含 agent-runner）。设计文档经多轮讨论修订，确立双层架构（tick 引擎 + GM/Agent）、Agent 工具范式、GM 生命周期、世界编辑器十面板等核心设计。

## M5-1：模拟器骨架（`d207697`）

simulator 工作区搭建：ECS 框架（Entity/Component/System）、tick 引擎主循环（10 秒间隔）、HTTP API 客户端、模板内容池发帖、概率性点赞/转发。editor 工作区搭建：Vite+React 框架 + 模拟器状态面板初版（端口 5174）。server 新增 admin 模块：管理端发帖（指定用户+时间戳）、批量关注、计数注水、admin key 认证（`dev-admin-key`）。

## M5-2：世界管理 + 设定文件库 + NPC 档案 + 快照（`bb2342c`）

server 世界管理扩展：元数据 PATCH、时钟控制（pause/resume/setScale/setTime）、轻量快照（better-sqlite3 db.backup + json，不复制 media，支持回滚）、世界复制、世界删除。设定文件库 lore CRUD（Markdown + 自动摘要提取）。NPC 人设档案 CRUD（npc-profiles.json）。预填内容批量导入 API。视频同源去重去除 `fs.linkSync` 硬链接，改为多行共享 file_name（不耦合 OS）+ `countByFileName` 引用计数。编辑器 Console / Lore / NPC Designer / Timeline 四个 Tab 面板。验证脚本 `scripts/verify-m5.ps1`。

## M5-3：话题系统 + 互动级联 + 内容池（`5dc8c68`）

migration v14 topics 表。话题 CRUD API（生命周期 emerging → fermenting → peak → declining → retired）。内容池 API（场景池 + 话题池，持久化 content-pools.json）。PostingSystem 改造为话题感知（兴趣 × 热度加权选题，优先话题池 → 场景池 → 兜底池）。CascadeSystem 互动级联（新帖扇出延迟反应，概率赞/转/回复，逐层衰减 ×0.5，最大深度 3）。编辑器新增 Topics + Pools 面板。

## M5-4：LLM Agent 接入（`e464dd9`）

LLMProvider 抽象接口 + Claude（@anthropic-ai/sdk）/ DeepSeek（openai SDK，兼容接口）/ Gemini（@google/genai）三家实现。12 个 Agent 工具定义（browse_timeline / read_lore / create_post 等）。AgentRuntime agentic 循环。LLMScheduler 优先级队列（urgent/normal/low + 并发控制 + 过期丢弃）。核心 NPC 发帖自动走 Agent，无配置回退模板，配置变更热生效。

server 端 AgentRunner：三家 provider 完整实现，直接操作世界数据库执行工具调用。多提供商配置体系：每个提供商独立配置（名称/来源/Base URL/API Key/模型列表），从 API 拉取模型列表；High/Low-tier 格式 `providerId|modelId`。Agent 执行日志含模型名称标注。

编辑器 LLM 面板：提供商管理 + 模型拉取与勾选 + High/Low-tier 下拉选择 + Agent 手动触发 + 执行日志查看。
