### 右边栏重构

- 新增公开接口 `GET /api/search/trends?limit=`（挂 search 模块）：统计近 7 个模拟日内帖子正文的 #话题 排行，返回 `{ trends: [{ tag, postCount }] }`。时间窗口用世界时钟（`clock.now()`）界定；SQL 以 `LIKE '%#%'` 预筛后在 service 层按与前端一致的正则 `/#[^\s#@]+/g` 解析；同帖同话题只计一次（验证：同帖重复话题计 1，软删除后从趋势消失）；按小写聚合、保留首现大小写。
- 新组件 RightSidebar 取代 Layout 内联右栏，自上而下：搜索框（列内吸顶）→ "有什么新鲜事"趋势卡（60 秒轮询，点击话题跳搜索页）→ "推荐关注"卡（复用 suggested-users 查询缓存，可直接关注）→ "当前世界"折叠卡 → 装饰性页脚（服务条款/隐私政策/Cookie 政策/©，无动作）。
- 世界卡默认折叠，折叠态只显示世界名与跳动模拟时钟，点击展开描述与流速；展开状态仅组件内存，刷新即回折叠。该改动源于用户反馈：常驻展开的世界信息"容易出戏"。
- 右栏 `overflow-y-auto` 可滚动并以 `.no-scrollbar` 工具类隐藏滚动条；首页空关注流的内联推荐关注组件保留未动。
- i18n 新增 `trends.*`、`footer.*` 双语文案。

### 浏览量（migration v5）

- posts 表新增 `view_count` 列（无回填，历史帖从 0 起算）。计数口径为曝光：帖子卡片 50% 进入视口算一次；同一页面会话内每帖只计一次，刷新后再看再计。
- 新增公开接口 `POST /api/posts/views`（optionalAuth，body `{ ids: 1–100 个唯一正整数 }`，返回 204）：批量曝光 +1，已删除与不存在的 id 静默忽略。挂 posts 模块——浏览无 (user, post) 行，不属于 interactions 的开关形态。
- 模拟器预留：虚拟用户将走同一上报接口；service 另留 `addViews(postId, delta)` 任意增量方法（内部走 `adjustCounts` 的 `view` delta），供未来上帝控制台/管理端批量注入模拟浏览量。
- 前端新增 `useViewTracking`：模块级共享 IntersectionObserver（threshold 0.5）+ 会话级已报 Set + 待报队列，2 秒 debounce 或满 20 条批量上报，页面隐藏时 `sendBeacon` 兜底；墓碑帖与引用嵌入卡不观察；详情页大卡走同一 PostCard 自然计入。世界热切换不刷新页面而帖子 id 空间重叠，`WorldsPage.activate()` 中调用 `resetViewTracking()` 清空会话状态。
- PostCard 互动行在赞与书签之间新增浏览量（`ri-bar-chart-2-line`，蓝色 hover，点击仅 stopPropagation 无动作，0 时只显示图标）。

### 全站计数格式化

- 新增 `client/src/i18n/formatCount.ts`：X 式缩写，截断一位小数、省无意义的 .0，整数运算避免浮点误差。zh：≥1万 → `x.x万`、≥1亿 → `x.x亿`；en：≥1K → `x.xK`、`x.xM`、`x.xB`。单位后缀按 locale 在函数内分支，不入 messages.ts。
- 套用位置：PostCard 四个互动计数（ActionButton 内部一处覆盖）、个人主页标题帖子数与关注/粉丝数、首页推荐关注粉丝数、右栏趋势卡帖子数、通知聚合文案的数字。Layout 未读徽标保持原有 99+ 截断不套用。

### 互动行定位修正

互动行由 `justify-between` 改为每个按钮（回复/转发/赞/浏览）各占一个 `flex-1` 左对齐单元格、书签贴行尾，图标横坐标不再随计数宽度移动（与 X 一致）。原互动行内联的删除按钮移除，并入新菜单。

### 帖子"…"菜单（migration v6）

- 帖子头部行尾新增"…"按钮（登录时显示），弹 X 样式下拉菜单。本人帖：删除（红色，沿用确认弹窗）、置顶/取消置顶；他人帖：关注/取消关注发帖人、屏蔽发帖人（红色，确认后执行）、隐藏这个帖子。
- **置顶**：`users.pinned_post_id` 列 + `POST/DELETE /api/posts/:id/pin`（仅本人帖，403 校验）。每用户一条，新置顶替换旧的；删除已置顶帖自动清除。个人主页"帖子"Tab 顶部带"已置顶"标记单独渲染置顶帖，后端用户时间线排除该帖防重复。
- **屏蔽**（单向隐藏）：新建 blocks 模块四件套，`POST/DELETE /api/users/:handle/block`。屏蔽时事务内自动取消屏蔽者对对方的关注（对方对屏蔽者的关注不动，对方全程无感）。被屏蔽者的帖子与转发从屏蔽者的关注流（latest/hot）、为你推荐、全站流、搜索、帖子回复区消失（转发条目同时校验转发者与原作者）；通知列表与未读数过滤被屏蔽 actor（含历史通知）；推荐关注排除。个人主页时间线、书签/喜欢列表、帖子详情主体不过滤（主动访问场景）。解除入口在被屏蔽用户的个人主页：关注按钮位置换为红边"解除屏蔽"按钮，简介区显示"你已屏蔽该用户"提示。
- **隐藏单帖**：`hidden_posts` 表挂 interactions 模块（与书签同构，无计数无通知），`POST/DELETE /api/posts/:id/hide`，过滤范围与屏蔽相同；无撤销 UI，取消接口留给模拟器。
- migration v6 一次包含 blocks 表、hidden_posts 表与 `users.pinned_post_id` 列。
- PostView 新增 `authorFollowedByViewer`（buildViews 批量补，菜单关注项的初始状态）；UserProfile 新增 `pinnedPostId` 与 `blockedByViewer`。
- 过滤实现为 repo 层 SQL `NOT IN` 子查询（匿名 viewer 跳过拼接），共用片段 `NOT_BLOCKED_AUTHOR`/`NOT_HIDDEN` 定义在 posts.repo 并被 timeline/search repo 引用；为你推荐与全站流的 repo 查询签名补了 viewerId 透传。
