## 时间轴设计校正（关键返工）

实施中两次偏离 `docs/m5-design.md`「时间轴面板（Premiere 范式）」，经用户指出后校正，并把教训固化为 memory `read-design-docs-first`：

1. **横轴语义**：从"固定起止铺满"改为**以当前模拟时间为分界的金色播放头**——左已发生、右待发生、随时钟滚动（设计原文）。
2. **块来源**：从"块=决策轨迹条"（只显示模拟器记录过的动作、漏种子/真人/历史）校正为**块=世界真实内容**——数据源改社交站全站流 `GET /api/timeline/global`（纯读 world.db、免鉴权、**独立于模拟器**，模拟器关也能用）；决策轨迹退为点开块后的"为什么"增强层（待 postId 合并）。
3. **取数策略**：global 流作"发现账号 + 顶层帖主轴 + 无限向后滚动"；按账号补拉回复（`?type=replies`）与转发（`/timeline` 的 type=repost，后被 T.1 统一端点取代）。

配套交互：无级缩放拉杆 + Ctrl+滚轮；轨道内同时段块**纵向错行堆叠**互不遮挡；账号显示头像 + 昵称（昵称直接取自流里每条的 displayName）；左上角时间为**可输入的自管文本框**（模拟时间，非系统时间），改值即跳转视图，目标早于已加载则自动向后补载；跨当前 24h 的刻度显日期；底部更明显的横向滚动条。路线图 / re-plan / index 同步把"块=轨迹条"措辞改回。

## 修复：WAL 只读读空

编辑器后端 `/api/trace` 返回 0 但 `sim-trace.db` 实有 2000+ 行：长期缓存的只读连接在 `-shm` 未就绪时（模拟器没起）打开会退化为"只读主库忽略 WAL"，而数据几乎全在 `-wal`。改为**每次查询新开只读连接、读完即关**，不缓存。

## T.1 互动上轴（路线图后续项，顺手做）

诊断"时间轴看不到点赞"：社交站无"带时间的互动事件"端点（`/likes` 丢了点赞时间）；但 `likes`/`reposts`/`follows` 三表均已有 `created_at`，**无需 migration**。新增统一互动事件端点：

- `shared` 加 `InteractionEvent`（like/repost/follow，带 `at`）。
- `interactions.repo.listUserActivity`：UNION 三表、按 `created_at` 倒序、游标 `[created_at, kind, ref]`（三段游标避免模拟器同 tick 多互动同毫秒翻页死循环）。
- `interactions.service.listUserActivity`：解析为事件（赞/转→帖预览，关注→被关注者），注入 usersService；`GET /api/users/:handle/interactions`（optionalAuth）。
- 编辑器后端代理 + renderer `LikeBlock`/`RepostBlock`/`FollowBlock`（灰色小条），转发改由此统一端点取（不再走 `/timeline`）；检视器分流赞/转/关注详情。
