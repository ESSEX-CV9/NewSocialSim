# 昵称 / ID / 头像生成

## 定位

批量创建虚拟账号时，须用确定性方式自动生成拟真的随机昵称（displayName）、ID（handle）与头像，不完全依赖 LLM。运行时纯数据 + 语法拼装、零 LLM；LLM 仅在离线扩充词库时可选使用（同"批量池化生成"一档）。核心账号由作者手写命名，氛围账号（大量、一次性）走自动生成。

## 硬约束：昵称与 ID 是两样东西

- `handle`（@ID）限 ASCII（`[a-zA-Z0-9_]{2,20}`），`displayName`（昵称）为任意 unicode。
- **handle 反映"注册时的身份"，基本不可改，故往往正经**；**displayName 反映"当下的状态"，可随时换、可玩 meme**。
- 二者**分开生成**，且默认**解耦**。最常见的真实形态正是「meme/抽象昵称 + 正经 ID」——如昵称「我是雷电将军的狗」配 ID `zhang_wei92`（当年用真名拼音注册、如今昵称玩梗）。因此 meme / 抽象风格的 handle 必须取自正经 handle 库（真名拼音 / 英文名 / 年份尾），不得从 meme 文本派生。

## 机制：风格分布 → 每风格语法 → 词库

与内容池同构：

1. **世界级风格分布**：声明该世界有哪些人群风格及权重，反映其人口风貌。如「现代中国」≈ 中文本名 50% / 诗词雅号 10% / meme 15% / 中英混 10% / 二次元 10% / 日系 5%；「international」≈ 美式 / 日系 / 韩系 / 中文各若干。
2. **每个风格 = 一套语法**，从词库抽槽拼装，分别产出 displayName 与（经派生规则）handle。

| 风格 | 昵称示例 | ID 示例 |
|---|---|---|
| 中文本名 | 林辰 / 苏曼 | `linchen` `+随机尾` |
| 诗词雅号 | 云深不知处 / 听风 | `tingfeng` |
| meme / 抽象 | 我是XX的狗 / XX退钱 | 正经库 `zhang_wei92` |
| 中英混 | 阿白Bai / Cino西诺 | `cino_x` |
| 美式 | Jake / sarah | `jake_miller` / `sarahxoxo` |
| 日系 / 中日混 | 奏 / yuki酱 | `kanade` / `yuki_n` |
| 韩系 | 민준 / 敏俊 | `minjun_` |
| 二次元 | 明日方舟阿米娅 | `amiya_ak` |

3. **词库 = 数据**（姓氏 / 名字常用字 / 诗词意境词 / meme 模板与角色名 / 英文 first+last / 日文罗马音 / 韩文 / 数字尾 / emoji / 罗马化映射表）。按三类布局存放：全局基础库入 git；世界级追加；随活动世界加载。

## ID 派生与去重

- handle 生成默认是其自身的事，从正经 handle 库抽，不强求贴昵称（真实用户常驴唇不对马嘴）。
- 仅本名类风格（中 / 日 / 韩本名）可选经罗马化表把 handle 关联到昵称（`林辰`→`linchen`）；需备一张罗马化映射库，属数据而非算法。
- handle 全站唯一，撞名即加数字 / 换尾重试。
- 生成结果必须自检、不踩代理建号守卫（`sim_` / `_amb` / `bot` / `npc` / 通用 stem+序号枚举）。

## 国际化

韩 / 日 / 美等用户即风格分布里的权重项。须提供若干世界预设：「现代中国」「东亚二次元圈」「international」「修真界」（换为道號 + 門派 风格的词库）。复制预设即换一整套人口风貌。

## 头像

按"零 LLM 优先"分档实施：

1. **兜底**：handle 哈希色块（网站已有）或 identicon 几何图——零成本、确定性。
2. **策展头像池**：世界备一组头像图（二次元脸 / 写实 / 插画），按风格派发（二次元号给动漫头像、美式号给写实），同优先媒体池范式。
3. **复用搜图管线**：按风格关键词走现有七源搜图拉头像入库。
4. **生成式**（扩散模型）：后置、可选。

近期先做第 1 与第 2 档（哈希 / identicon 兜底 + 策展头像池）；搜图与生成式后置。

## 接入点

生成器插入批量代理建号流程：氛围账号按世界风格分布自动生成昵称 + ID + 派发头像；核心账号由作者手写。生成器与内容池共享同一思路——昵称即一种按风格语法从词库拼装的内容。
